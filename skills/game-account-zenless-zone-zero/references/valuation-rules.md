# 绝区零（ZZZ）估值规则

updated_at: 2026-05-23

## 社区证据要求

评分前读取 `references/community-evidence.md` 和 `references/agent-knowledge.md`。真实买号筛选默认只信任 7 天内的社区证据；若快照已满 7 天、跨版本、或账号中出现未覆盖的新限定代理人/音擎，`confidence` 最高只能为 `medium`，并输出规则刷新建议。

## 最高优先级规则

不要把 S 代理人总数、绳匠等级或常驻 S 高影画当作主价值来源。必须先判断：

1. 限定 S 代理人是否命名且符合用户目标。
2. 专属音擎是否匹配核心代理人。
3. 是否能组成可用队伍，而不是孤立角色堆叠。
4. 菲林/母带/邦布券等资源是否足够。
5. HoYoverse、邮箱交付/实名、PSN、TAP、B服/渠道服和找回风险是否清楚。

若用户指定“全部虚狩”或类似硬条件，当前按星见雅、仪玄、叶瞬光三位虚狩处理。三虚狩不能只检查“角色都在”，必须继续检查能否拆成三支互不抢核心队友的可用队伍；“三虚狩 + 两个泛用辅助”只能算队友不足，不得高置信推荐。

平台详情若提供 `agentStatuses` / `game_assets.agent_statuses`，必须用它确认代理人影画；专属音擎需要做双源校验。螃蟹/盼之绝区零资产卡角标的格式为 `x` 或 `x+y`：`x` 表示影画/命座，`+y` 表示该代理人的专属音擎数量。若角标只有 `x`，不能直接判定无专武，必须同步读取页面 `S级音擎` / `S级武器` 名称清单，并用 `references/signature-engines.json` 逐名匹配该角色专武。匹配成功时可确认专武；既没有 `+y` 又没有可匹配音擎名时，才按“专武未确认”处理。不要用账号总 S 音擎数量或标题里的“几命”推断核心代理人带专武。

## 可执行评分框架

```yaml
score_weights:
  limited_agent_score: 0-35
  engine_fit_score: 0-15
  team_completeness_score: 0-20
  resource_score: 0-15
  progression_score: 0-5
  price_fit_score: 0-10
  risk_penalty: 0-35
  missing_data_penalty: 0-25
```

### 限定代理人

- 高价值限定 S 代理人：每个 7-9 分。
- 中价值或快照未覆盖限定：每个 3-5 分并降置信度。
- 常驻 S：每个 1-3 分；高影画最多额外 1-2 分，不能突破限定核心权重。

### 音擎

- 核心限定代理人带专属音擎：每个 5-6 分。
- S 音擎适配中价值代理人：每个 2-3 分。
- 泛用或不适配 S 音擎：每个 0-1 分。
- 当 `agent_statuses["代理人"]` 为 `x+y` 且 `y > 0` 时，把该代理人计入“带专属音擎”。
- 当角色角标只有 `x` 时，检查 `sWEngineNames` / `game_assets.s_w_engine_names` / `game_assets.w_engines[].name`，并用 `references/signature-engines.json` 的本地专武表确认归属；名称精确匹配到该角色专武时，同样计入“带专属音擎”。
- 只有总 S 音擎数量、泛用“专武多”文案、或音擎名称无法匹配本地专武表时，最多按人工确认项处理。

### 队伍完整度

- 至少一队定位完整且含限定核心：10-14 分。
- 两队可用且资源/音擎支撑：15-20 分。
- 孤立限定或缺击破/支援/防护/异常支撑：3-8 分。
- 只有 S 数量无定位：0-3 分。

### 三虚狩三队完整性

当用户硬性要求全部虚狩时，`team_completeness_score` 必须先做“独立三队”检查：

```yaml
void_hunter_cores:
  - 星见雅
  - 仪玄
  - 叶瞬光
preferred_team_slots:
  星见雅:
    preferred: [浮波柚叶 + 南宫羽, 浮波柚叶 + 莱卡恩, 浮波柚叶 + 苍角]
    alternatives: []
  仪玄:
    preferred: [卢西娅 + 橘福福, 卢西娅 + 琉音]
    alternatives: []
  叶瞬光:
    preferred: [照 + 耀嘉音, 照 + 琉音]
    alternatives: []
```

判定规则：

- 三队完整：三位虚狩都在，且每位虚狩能分配到两名不与其它虚狩重复占用的适配队友，给 18-20 分。
- 两队完整：三位虚狩都在，但只能无重复拆出两支完整队，给 11-14 分，并列入人工确认。
- 共享辅助陷阱：三位虚狩都在，但关键辅助总数不足，例如只有琉音、耀嘉音、照、千夏中的两三位，不能按三队完整评分。
- 旧口径陷阱：星见雅 + 柚叶 + 月城柳、叶瞬光 + 照 + 千夏、仪玄 + 卢西娅 + 橘福福等组合不能自动按当前三支虚狩队完整处理；必须优先按上表核对雅队、叶队和仪玄队。
- 平台标题只写 S 数量、黄数或“三虚狩”但没有列出队友时，`confidence` 最高为 `medium`，必须要求卖家补角色页截图。
- 螃蟹/盼之详情 adapter 能返回 `agentStatuses` 时，三虚狩影画以该字段为准；专属音擎优先用 `x+y`，角标只有 `x` 时必须再用 `sWEngineNames` 和 `references/signature-engines.json` 交叉确认。若 adapter 未返回 `agentStatuses` 或 S 音擎名称清单但页面应有资产卡片，`confidence` 最高为 `medium`。
- 柚叶是星见雅队硬优先信号；雅队优先按 `雅 + 柚叶 + 南宫羽/莱卡恩/苍角` 判断。没有柚叶时，不得默认星见雅队完整。
- 叶瞬光队优先按 `叶瞬光 + 照 + 耀嘉音/琉音` 判断；只有照 + 千夏时按旧口径/下位可玩处理，不算当前三虚狩完整队。
- 仪玄队优先按 `仪玄 + 卢西娅 + 橘福福/琉音` 判断；只有仪玄本体或只有泛用支援不算完整。

### 舒适度与额外队伍加分

三虚狩硬条件满足后，再用舒适度信号排序；这些信号是加分项，不能替代三虚狩、队友、资源和风险硬检查。

```yaml
comfort_bonus_signals:
  void_hunter_2_plus_1:
    rule: 星见雅/仪玄/叶瞬光任一达到 2+1 加分；三位都 2+1 明显加分
  extra_team_core_0_plus_1:
    rule: 剩余队伍的对应主 C 或核心达到 0+1 加分
  astra_yao_1_plus_1:
    rule: 耀嘉音/耀佳音 1+1 是舒适度加分项
  direct_electric:
    required: [希希芙, 席德, 耀嘉音]
  anomaly_release:
    required: [千夏, 爱芮, 南宫羽]
  vivian_disorder:
    required: [薇薇安, 紊乱搭档]
```

- 直伤电队按 `希希芙 + 席德/希德 + 耀嘉音/耀佳音` 验证；缺任一位只算待确认，不把标题里的“电队”当成完整队。
- 异放队按“妄想天使三小只”验证，当前买号语境按 `千夏 + 爱芮 + 南宫羽` 处理；缺任一位只能作为风险备选或收藏项未满。
- 薇薇安紊乱队是额外加分项；至少需要薇薇安 + 一名明确紊乱/异常搭档（如简、柏妮思、月城柳、爱丽丝、格莉丝等）。薇薇安 `0+1` 或搭档 `0+1` 可作为舒适度加分，但不能替代三虚狩硬条件。
- `agentStatuses` 中 `2+1`、`1+1`、`0+1` 比标题文案更可靠；若只有 `2`、`1`、`0`，必须继续查 S 级音擎名称清单。标题只写“带签”“专武多”但无法归属时，只能列人工确认。

### 资源

高价值资源包括：

- 菲林
- 菲林底片
- 加密母带
- 原装母带
- 邦布券
- 余波信号
- 丁尼和突破/技能材料

抽卡资源折算口径：

- `estimated_pulls = (菲林 + 菲林底片) / 160 + 加密母带 + 原装母带 + 邦布券`。
- `菲林底片` 是菲林数值字段，不是 1:1 抽卡券；不要把 140 菲林底片误算成 140 抽。
- 卖家备注里的“还有 N 抽”“垫 N 抽”只能作为辅助证据；若与字段折算差异明显，列入人工确认，不直接给高资源分。

资源可支撑后续版本抽卡给 10-15 分；资源少或未披露给 0-5 分。

### 风险扣分

- HoYoverse/米哈游通行证绑定或实名不明：10-20 分。
- 邮箱交付状态：
  - 邮箱未绑定：低找回风险，额外加 4-5 分。
  - 邮箱未实名出售：低找回风险，额外加 3-4 分；在同等资产下应优先于“邮箱实名出售”“邮箱不出售”。
  - 邮箱实名出售：不直接排除，但必须人工确认邮箱实名、换绑和找回链路，扣 3-6 分。
  - 邮箱不出售：高找回风险，扣 10-16 分；低风险偏好下只作备选。
  - 邮箱已注销或邮箱状态不明：扣 6-10 分并列入人工确认。
- PSN 已绑定或状态不明：8-18 分。
- TAP/B服/渠道服不符合用户要求：10-25 分。
- 无找回包赔或官方验号：8-15 分。
- 代理人/音擎/资源/邮箱/绑定缺失：每项 5-12 分。

## 排序硬规则

- “常驻 S 高影画 + S 数量多 + 音擎/绑定不明”的账号不得排在“限定核心 + 专属音擎 + 队伍/资源/风险明确”的账号前。
- 同等资产和平台保障下，“邮箱未绑定”或“邮箱未实名出售”的账号应排在“邮箱实名出售”“邮箱不出售”“邮箱状态不明”的账号前。
- 只写 S 数量或绳匠等级的账号不能进入 Top 1。
- 用户硬性要求全部虚狩时，缺任一虚狩的账号直接排除；三虚狩齐但无法验证独立三队的账号只能作为风险备选，不得压过能组成三队的账号。
- 若用户给定预算内没有满足硬条件且三队完整的账号，应扩大到 `flex_budget`，优先寻找“价格最低且硬条件完整”的账号；不能用预算内但不满足三队的账号充当主推荐。
- 低风险偏好下，HoYoverse/PSN/TAP 绑定不明账号应排除或只作备选。
- 代理人、音擎、资源、邮箱交付/实名、HoYoverse/PSN/TAP 绑定或官方验号缺失时，必须列入人工确认，不能用资产分自动补齐。
- 对螃蟹/盼之的绝区零账号，缺少 `agent_statuses` 或 `s_w_engine_names` / `sWEngineNames` 时不得高置信判断“全部虚狩都有对应专武/辅助完整”；必须回到详情页资产卡片和 S 级音擎页复核或要求截图。

## 推荐解释要求

每个推荐账号必须说明：

- 哪些代理人是真正限定核心。
- 专属音擎是否匹配核心代理人。
- 队伍是否完整，缺什么定位。
- 资源是否足够后续抽卡。
- 邮箱交付/实名、HoYoverse/PSN/TAP/B服/找回风险是否可接受。
- 账号相对社区证据快照是强一致、部分一致、弱一致还是冲突。
